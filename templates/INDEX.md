## Como Adicionar Novos Templates

### Nova Máscara

1. Criar arquivo em `mascaras/` com nome `tc-[regiao]-[contraste].md` ou `angio-[regiao].md`
2. Adicionar frontmatter YAML com:
   - `tipo`: tipo do exame (ex: `tc-abdome`, `tc-torax`)
   - `contraste`: `com` ou `sem`
   - `subtipo`: opcional (ex: `tep`, `idoso`)
   - `urgencia_padrao`: `true` ou `false`
   - `palavras_chave`: array de palavras-chave para matching
3. Estrutura do laudo:
   - Título (maiúsculo)
   - `<!-- OPCIONAL:urgencia -->` - Urgência (remover se eletivo/ambulatorial)
   - `<!-- OPCIONAL:indicacao -->` - INDICAÇÃO (inserir se mencionada)
   - TÉCNICA
   - ANÁLISE com regiões delimitadas por `<!-- REGIAO:nome -->` e `<!-- /REGIAO:nome -->`
4. Ordem de inserção no início da ANÁLISE:
   - `<!-- REGIAO:comparativo -->` (apenas modo comparativo)
   - `<!-- REGIAO:dispositivos -->`
   - `<!-- REGIAO:cirurgias -->`
   - Demais regiões anatômicas
6. Atualizar este INDEX.md

### Novo Achado

1. Criar arquivo em `achados/[orgao]/` com nome descritivo em kebab-case
2. Adicionar frontmatter YAML com:
   - `regiao`: região anatômica (ex: `rim`, `figado`, `torax`)
   - `palavras_chave`: array de palavras-chave para matching
   - `requer`: array de campos obrigatórios (ex: `["medida", "lado"]`)
   - `opcional`: array de campos opcionais
   - `medida_default`: valor padrão para medida se não fornecido
3. Usar placeholders `{{nome}}` para valores variáveis
4. Atualizar este INDEX.md

## Notas Importantes

- Todos os templates usam formatação HTML hardcoded (não gerada pelo LLM)
- Placeholders são substituídos pelo LLM baseado no texto ditado
- Máscaras filtradas por contexto do exame (tipo, contraste, subtipo) usando regex
- Achados filtrados por região relevante ao tipo de exame
- Dispositivos e cirurgias podem aparecer em qualquer tipo de laudo
- Modo comparativo requer laudo anterior e data anterior no texto ditado
